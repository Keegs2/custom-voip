1. Remote Call Forwarding (RCF)

RCF lets you assign a phone number (DID) that automatically forwards to any destination. Update the forwarding destination instantly via API or through the web dashboard — no carrier coordination required. Changes take effect within seconds.

Endpoints
GET /rcf List all RCF numbers
POST /rcf Create an RCF number
GET /rcf/{did} Get RCF details
PUT /rcf/{did} Update forwarding destination
DELETE /rcf/{did} Remove an RCF number
Example — Create an RCF number
POST /rcf { "customer_id": 1, "did": "+12125551234", "forward_to": "+18005559999", "pass_caller_id": true, "ring_timeout": 30 }
Example — Change where a number forwards
PUT /rcf/+12125551234 { "forward_to": "+13475551111" }
2. API Calling (Programmable Voice)

Initiate and control calls programmatically. When a call connects, your webhook URL receives events and you respond with instructions to control the call flow. This enables IVR systems, call queues, conference bridges, and any custom voice application.

Endpoints
POST /calls Initiate an outbound call
GET /calls/{call_id} Get call status
POST /calls/{call_id}/update Modify a live call (hangup, transfer)
Example — Make an outbound call
POST /calls { "from_did": "+15553001001", "to": "+18005559999", "webhook_url": "https://your-server.com/voice", "timeout": 60 }
Response
{ "call_id": "uuid-here", "status": "initiated", "tier": "api_basic", "per_call_fee": 0.01 }
Webhook Events

Your webhook receives POST requests with call events. Respond with XML to control the call:

<Say> Text to speech — speak text to the caller
<Play> Play an audio file from a URL
<Gather> Collect DTMF (keypad) input from the caller
<Dial> Connect to another number or SIP endpoint
<Hangup> End the call
Example — Hangup a live call
POST /calls/{call_id}/update { "action": "hangup" }
3. SIP Trunking

Connect your PBX or SBC directly to our platform via SIP. Authenticate with IP address. CPS and Call Paths are configured independently — CPS controls call setup rate, while Call Paths control how many simultaneous calls your trunk can support at once.

Endpoints
GET /trunks List trunks
POST /trunks Create a trunk
GET /trunks/{id} Get trunk details
PUT /trunks/{id} Update trunk settings
GET /trunks/{id}/stats Real-time trunk statistics
GET /trunks/{id}/ips List authorized IPs
POST /trunks/{id}/ips Add an authorized IP
DELETE /trunks/{id}/ips/{ip_id} Remove an authorized IP
GET /trunks/call-paths List call path packages
PUT /trunks/{id}/call-paths Assign a call path package to a trunk
Example — Create a trunk
POST /trunks { "customer_id": 3, "trunk_name": "Main Office PBX", "max_channels": 50, "cps_limit": 5, "auth_type": "ip" }
Example — Add an authorized IP
POST /trunks/1/ips { "ip_address": "203.0.113.50", "description": "Office PBX" }
4. CDR (Call Detail Records)

Query call history with flexible filters. CDRs include duration, billing, and routing details. Use the summary endpoint for aggregated statistics across a date range.

Endpoints
GET /cdrs Query CDRs with filters
GET /cdrs/summary Aggregated CDR statistics
GET /cdrs/{uuid} Get a single CDR by UUID
Example — Query recent CDRs
GET /cdrs?customer_id=1&limit=10&start_date=2026-04-01T00:00:00
5. Customer Management

Manage customer accounts, balances, and credit. Customers are the top-level billing entity — all trunks, RCF numbers, and API DIDs are associated with a customer.

Endpoints
GET /customers List customers
POST /customers Create a customer
GET /customers/{id} Get a customer
PUT /customers/{id} Update customer details
GET /customers/{id}/balance Get balance and credit info
POST /customers/{id}/credit Add credit to account
6. Rate Limits & Billing

CPS (Calls Per Second) limits are enforced per account tier. Call Paths for SIP trunks are purchased separately and control concurrent call capacity.

API Calling Tiers
TierCPSMonthly FeePer-Call Fee
API Basic 5 CPS $49/mo $0.0100
API Standard 8 CPS $149/mo $0.0080
API Premium 15 CPS $399/mo $0.0050

TierCPSMonthly Fee

Call Path Packages (SIP Trunks)

Call paths are purchased per-trunk and control concurrent call capacity. CPS and call paths are independent — a 5-CPS trunk with 100 call paths can handle 100 simultaneous calls at up to 5 new calls per second.

CPS Limit Exceeded

When your CPS limit is exceeded, the API returns 429 Too Many Requests with a response body that includes upgrade recommendations:

{ "detail": "CPS limit exceeded", "current_tier": "api_basic", "cps_limit": 5, "upgrade_to": "api_standard" }
